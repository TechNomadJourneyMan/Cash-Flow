# Digital Central Bank & Global Exchange — Firebase-портал

Realtime-портал на **Firebase (Firestore + Anonymous Auth)** с тремя ролями: **админ / банкир / игрок**. Игроки заходят с разных телефонов по коду комнаты, всё синхронизируется мгновенно.

Файлы:
- `firebase-portal.html` — само приложение (один файл, Firebase Web SDK через CDN, без сборки).
- `firestore.rules` — правила безопасности Firestore.
- `FIREBASE-SETUP.md` — этот файл.

---

## 1. Создать проект Firebase (5 минут)

1. Откройте https://console.firebase.google.com → **Add project** → задайте имя (например `cashflow-bank`). Google Analytics можно выключить.
2. В проекте: **Build → Firestore Database → Create database** → режим **Production** → выберите регион (ближе к вам, напр. `eur3`/`europe-west`).
3. **Build → Authentication → Get started → Sign-in method → Anonymous → Enable → Save.**
   (Приложение входит анонимно «под капотом» — игрокам ничего вводить не нужно, кроме имени.)

## 2. Получить конфиг и вставить в приложение

1. В консоли: **Project settings (шестерёнка) → General → Your apps → Web (`</>`)** → зарегистрируйте веб-приложение (Hosting можно не включать сейчас).
2. Скопируйте объект `firebaseConfig` — он выглядит так:
   ```js
   const firebaseConfig = {
     apiKey: "AIza...",
     authDomain: "cashflow-bank.firebaseapp.com",
     projectId: "cashflow-bank",
     storageBucket: "cashflow-bank.appspot.com",
     messagingSenderId: "1234567890",
     appId: "1:1234567890:web:abc123"
   };
   ```
3. Откройте `firebase-portal.html`, найдите вверху блок **`/* === FIREBASE CONFIG === */`** и вставьте свои значения вместо плейсхолдеров `PASTE_...`.

> `apiKey` в веб-приложениях Firebase **не секрет** — это идентификатор проекта, а не пароль. Доступ ограничивают правила Firestore (`firestore.rules`).

## 3. Залить правила безопасности

**Вариант A — через консоль (просто):**
Firestore Database → вкладка **Rules** → вставьте содержимое `firestore.rules` → **Publish**.

**Вариант B — через Firebase CLI (рекомендуется для деплоя):**
```bash
npm i -g firebase-tools
firebase login
cd "/Users/ansarisenoff/Cash Flow"
firebase init firestore        # выберите проект, согласитесь с firestore.rules
firebase deploy --only firestore:rules
```

## 4. Запустить

**Локально (тест на одном ноутбуке + телефоны в той же сети):**
Просто откройте `firebase-portal.html` в браузере (двойной клик). Realtime уже работает через облако Firestore. Чтобы зайти с телефонов, нужен общий URL — см. деплой ниже (открытие `file://` на телефоне не подойдёт).

**Деплой на Firebase Hosting (чтобы заходить с телефонов по ссылке):**
```bash
firebase init hosting          # public dir: оставьте текущую папку или скопируйте html в ./public как index.html
# при вопросе про single-page app — можно "No"
firebase deploy --only hosting
```
Получите ссылку вида `https://cashflow-bank.web.app` — её показываете игрокам на проекторе.

> Альтернатива хостингу: Vercel/Netlify (перетащить файл), GitHub Pages, или любой статический хостинг. Это статический HTML — backend не нужен, всё общение идёт напрямую с Firestore.

## 5. Регистрация, вход и роли

Роли в портале: **Админ (гейм-мастер) · Банкир · Аудитор · Игрок**.

**Регистрация / вход участника (единый экран):**
- Игрок вводит **код комнаты + своё имя + личный PIN** (4 цифры) и жмёт «Войти / Зарегистрироваться».
- Если имени ещё нет в комнате — создаётся новый профиль (игрок) с этим PIN и пустым финансовым бланком.
- Если имя уже есть — выполняется вход по PIN (роль берётся из профиля: игрок / банкир / аудитор).
- PIN хранится в профиле и синхронизируется, поэтому участник заходит с любого устройства по «имя + PIN».

**Вход админа:** отдельная форма — код комнаты + PIN админа (задаётся при создании комнаты).

**Назначение ролей (делает админ):** Игроки → у игрока кнопка **«👤 Роль»** → выбрать `Игрок / Банкир / Аудитор`, а также назначить **персонального аудитора** (кто проверяет именно его бланк — по правилам это «игрок справа»). Роль применяется **мгновенно** на устройстве участника (через realtime), повторный вход не нужен.

**Эффективная роль — живая:** UI участника определяется ролью в его профиле. Повысили игрока до банкира/аудитора — у него тут же появляются соответствующие вкладки.

## 6. Финансовый бланк игрока (CashFlow-бланк)

У каждого участника есть цифровой бланк (вкладка **📝 Бланк** у игрока):
- **Доходы:** зарплата, проценты, дивиденды, недвижимость, бизнес.
- **Расходы:** налоги, ипотека, кредит на авто, кредитные карты, розничные, кредит банка, прочие, дети (кол-во × расход на ребёнка).
- **Активы:** сбережения, акции/фонды, недвижимость, бизнес (динамические строки).
- **Пассивы:** ипотека, кредиты, долги.
- **Считается автоматически:** общий доход, общий расход, пассивный доход, **денежный поток (получка)**.

**Аудит-процесс:** игрок заполняет бланк → «📤 На проверку» → статус `на проверке`. **Аудитор / банкир / админ** открывают бланк, могут **редактировать** и нажать **«✓ Подтвердить»** или **«Отклонить»** с заметкой. Игрок видит результат и заметку в реальном времени. Админ/банкир также могут начислить рассчитанный денежный поток в кошелёк кнопкой **«💱 Поток»**.

## 7. Как пользоваться в игре

1. **Админ** → «Создать комнату» → код, баланс, столы, PIN админа/банкира → показывает **код + ссылку** на проекторе.
2. **Участники** заходят с телефонов: код + имя + личный PIN → выбирают стол/команду → кошелёк и бланк.
3. **Админ** назначает банкиров и аудиторов (кнопка «Роль») и при желании персонального аудитора каждому игроку.
4. **Игроки** заполняют бланки, переводят деньги, торгуют, создают пулы; **банкиры** подтверждают крупные операции и «День получки»; **аудиторы** проверяют бланки; **админ** ведёт экономику (события, биржа, кризисы, фазы).

Все данные сессии хранятся в Firestore и **синхронизируются у всех участников в реальном времени**.

## 8. Модель данных (Firestore)

```
rooms/{ROOMCODE}                 — комната: name, phase, round, mode, trendId,
                                   riskIndex, centralBankRate, economy{}, settings{},
                                   assets[], teams[], tables[], counters{}, achievements[]
  ├─ users/{uid}                 — участники: name, pin, role (player/banker/auditor),
  │                                tableId, teamId, status, balance, tags[], portfolio{},
  │                                auditorId (кто проверяет бланк), sheet{} (фин. бланк), online
  ├─ transactions/{id}           — все денежные операции (атомарно, с balanceBefore/After)
  ├─ loans/{id}                  — кредиты: principal, outstanding, rate, status
  ├─ pools/{id}                  — инвестиционные пулы (+ participants[])
  ├─ startups/{id}               — венчурные заявки
  ├─ offers/{id}                 — предложения инвесторов
  ├─ approvals/{id}              — очередь подтверждений банкира
  ├─ news/{id}                   — лента новостей/событий
  ├─ history/{id}                — история партии
  └─ audit/{id}                  — audit log действий
```
Сущности 1:1 соответствуют разделу «Модель данных» (§31) ТЗ. Команды/столы хранятся массивами в документе комнаты (они небольшие) — это упрощает реактивность.

## 9. Безопасность: MVP vs продакшен

- **Сейчас (MVP):** вход анонимный, роли и PIN-ы проверяются в приложении; правила пускают любого аутентифицированного клиента к данным комнаты. Этого достаточно для игры в одном зале с доверенными людьми.
- **Продакшен-хардненинг (рекомендуется при публичном доступе):**
  - PIN-ы и роли — через **Custom Claims** + **Cloud Functions** (нельзя доверять клиентским проверкам).
  - Денежные операции (`changeMoney`, `transfer`, `applyEvent`, `payday`) переносятся в **Cloud Functions** с серверной валидацией и Firestore-транзакциями — клиент не пишет балансы напрямую.
  - Правила Firestore: запись только через функции; чтение — по `roomId` и роли.
  - App Check (reCAPTCHA) против ботов.

## 10. Частые проблемы

| Симптом | Причина / решение |
|---|---|
| «Missing or insufficient permissions» | Не залиты `firestore.rules` или не включён Anonymous Auth. |
| Пустой экран, в консоли ошибка про `firebaseConfig` | Не вставлен конфиг (остались `PASTE_...`). |
| Игроки на телефонах не видят комнату | Открыли `file://` локально. Нужен общий URL — задеплойте на Hosting/Vercel. |
| Данные не обновляются между устройствами | Проверьте, что все вошли в **один и тот же код комнаты** (регистр не важен — код приводится к верхнему). |
| Хочу начать заново | Админ → Настройки → «Сбросить комнату» (удаляет документы комнаты), либо удалите коллекцию в консоли Firestore. |
